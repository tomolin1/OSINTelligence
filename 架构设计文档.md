> 版本: 1.0 | 2026年7月 | 架构与数据规范组

## 一、概述

### 1.1 项目背景

随着地缘政治冲突加剧和网络攻击活动频发，APT（高级持续性威胁）组织持续对政府、军事、能源、金融等关键基础设施发动攻击。传统威胁情报收集依赖人工分析，存在效率低、覆盖面窄、时效性差等问题。大语言模型（LLM）在实体识别、关系抽取、知识整合等方面的突破性进展，为自动化威胁情报分析提供了全新的技术路径。

本项目旨在设计并实现一套基于大模型的开源威胁情报APT研判系统，能够自动从多源异构情报数据中提取 APT 组织、恶意软件、漏洞、攻击技术等关键实体及其关联关系，构建知识图谱，并生成结构化威胁分析报告。

### 1.2 设计目标

1. 实现多源情报数据的自动化采集与清洗
2. 基于大模型实现实体识别（NER）和关系抽取（RE）
3. 构建 APT 威胁知识图谱，支持多维查询与可视化
4. 支持 3 个目标组织（APT28/Lazarus/Kimsuky）的深度分析
5. 生成结构化威胁分析报告

### 1.3 架构设计原则

- **模块化**：各模块职责清晰，接口契约驱动，松耦合设计
- **契约优先**：schema.json 定义全系统数据格式，一经锁定禁止私自修改
- **容错设计**：LLM调用支持 try...except 重试，批处理支持断点续传
- **可扩展**：支持新增情报源、新增实体类型、新增关系类型

### 2.3 数据流

```
爬虫采集原始文本 → 统一JSON格式 → LLM抽取实体关系 → 存入Neo4j → REST API查询 → 前端渲染
```

## 三、模块详细设计

### 3.1 数据采集模块

负责从多源开源情报渠道自动采集数据，支持来源类型：威胁报告、CVE漏洞库、CERT安全公告、安全博客、新闻网站、Twitter/X安全频道、Telegram安全群组。

输出格式统一为 JSON，核心字段：`source_url`, `source_type`, `title`, `raw_text`(纯文本), `language`, `crawl_time`, `metadata`。

文本清洗规则：
- 去除 HTML/XML 标签、CSS、JavaScript
- 去除空白字符和不可见字符
- 统一换行符和缩进

### 3.2 LLM抽取模块

核心模块，负责从爬虫产出的纯文本中自动识别威胁情报实体及其关系。

**实体识别（NER）支持11种类型**：
| 类型 | 示例 |
|------|------|
| APT_GROUP | APT28, Lazarus, Kimsuky |
| MALWARE | LameHug, X-Agent, WannaCry |
| TOOL | Metasploit, Cobalt Strike |
| CVE | CVE-2025-12345 |
| TECHNIQUE | 鱼叉式钓鱼(T1566) |
| PERSON | 黑客、研究员 |
| COUNTRY | Russia, North Korea |
| INDUSTRY | 政府机构、金融 |
| CAMPAIGN | 乌克兰网络攻击行动 |
| ORGANIZATION | GRU, Mandiant |
| TIME | 2025年3月 |

**关系抽取（RE）支持10种类型**：
| 类型 | 含义 | 示例 |
|------|------|------|
| ATTRIBUTED_TO | 归因于 | APT28 → GRU |
| USES | 使用 | APT28 → LameHug |
| TARGETS | 瞄准 | APT28 → Ukraine |
| DEVELOPS | 开发 | Lazarus → WannaCry |
| EXPLOITS | 利用漏洞 | APT28 → CVE-2025-12345 |
| RELATED_TO | 通用关联 | — |
| AFFILIATED_WITH | 组织间关联 | Kimsuky ↔ APT43 |
| LEADS_TO | 攻击链顺序 | T1566 → T1203 |
| DISCOVERED_BY | 被发现 | APT28 → Mandiant |
| EMPLOYS | 雇佣 | GRU → 某人员 |

**Prompt工程设计**：
- System Prompt 强制要求100%输出合法JSON
- 使用 Few-shot 示例引导输出格式
- 实体和关系分别抽取，降低单次任务复杂度
- 必须包裹 try...except，非JSON输出时重试
- 输出严格验证 schema，字段缺失时补调

### 3.3 图谱存储模块

采用 Neo4j 图数据库，原生支持属性图模型，适合存储和查询复杂的实体关系网络。

共定义9种节点标签：APT_GROUP, MALWARE, TOOL, CVE, TECHNIQUE, PERSON, COUNTRY, INDUSTRY, CAMPAIGN

共定义10种关系类型，每条关系附带：`id`, `confidence`, `source_text`, `mitre_attack_id`(可选)

索引策略：
- APT_GROUP: name, origin_country
- MALWARE: name
- CVE: name
- TECHNIQUE: id, name
- COUNTRY: name, code
- INDUSTRY: name

### 3.4 后端服务模块

共5大类10个接口：

| 类别 | 接口 | 方法 | 说明 |
|------|------|------|------|
| 分析 | /api/v1/analyze | POST | 单文本分析 |
| 分析 | /api/v1/analyze/file | POST | 文件分析 |
| 分析 | /api/v1/analyze/batch | POST | 批量分析 |
| 图谱 | /api/v1/graph | GET | 获取图谱数据 |
| 图谱 | /api/v1/graph/expand/:id | GET | 展开节点 |
| 图谱 | /api/v1/graph/timeline | GET | 时间轴过滤 |
| 实体 | /api/v1/entities/search | GET | 搜索实体 |
| 实体 | /api/v1/entities/:id | GET | 实体详情 |
| 报告 | /api/v1/report | GET | 生成报告 |
| 系统 | /api/v1/health | GET | 健康检查 |

### 3.5 前端展示模块

技术栈：React 18 + ECharts-for-React

核心功能：
- 搜索交互：搜索框输入关键词，返回实体列表
- 图谱渲染：ECharts 力导向图展示节点和关系，按类型着色
- 悬停详情：鼠标悬停节点显示实体信息
- 展开节点：点击节点加载更多关联（爆炸图）
- 时间轴：拖动滑块过滤时间范围，展示攻击演变
- 报告生成：一键生成威胁分析报告

渲染优化：
- 默认只显示2层以内的关系
- 深层关系通过「点击加载更多」实现
- 节点数超过200时启用 ECharts 的 roam 缩放

## 七、安全设计

**LLM调用安全**：
- API Key 通过环境变量注入，不硬编码在代码中
- 请求限流（429状态码保护）
- 输出通过 JSON Schema 校验
- 断点续传避免重复调用

**数据安全**：
- 文件格式校验：仅接受 .json 和 .txt
- 文件大小限制：不超过10MB
- 路径防护：禁止目录遍历

**API安全**：
- 输入参数严格校验（类型、范围、必填项）
- 统一错误处理，不泄露内部细节

## 八、开发规范

**代码规范**：Python 遵循 PEP 8，JavaScript 遵循 Standard 规范

**协作规范**：
- schema.json 为全系统数据契约，修改需架构组审核
- 每个接口修改需同步更新 api_docs.md
- Mock 数据与 Schema 保持一致
- 黄金测试集中维护，LLM组和测试组每日比对
